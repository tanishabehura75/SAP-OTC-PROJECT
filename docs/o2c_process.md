## Step 1 — Sales Inquiry (VA11)

### What It Is
A Sales Inquiry is the first formal touchpoint in the O2C cycle. It records a customer's interest in purchasing a product, capturing their questions around availability, pricing, and delivery timelines. It is a non-binding document — no commitment is made by either party at this stage.

### Navigation Path
`SAP Easy Access → Logistics → Sales and Distribution → Sales → Inquiry → Create`

### T-Code
`VA11`

### Key Inputs

| Field | Value |
|---|---|
| Inquiry Type | IN |
| Sales Organization | PE01 |
| Distribution Channel | 10 |
| Division | 00 |
| Sold-to Party | CUST-1001 (TechNova Enterprises) |
| Material | MAT-PB15 (ProBook Laptop 15) |
| Requested Quantity | 20 EA |
| Validity From | (Current Date) |
| Validity To | +30 Days |

### Output
- **Inquiry Document Number** generated by the system (e.g., 10000001)
- Stored in SAP as a reference for quotation creation

### Screenshot Reference
`01_VA11_inquiry_creation.png`

## Step 2 — Sales Quotation (VA21)

### What It Is
A Sales Quotation is a formal commercial offer sent to the customer. It confirms the pricing, product specifications, and terms under which PixelEdge is willing to supply the goods. Unlike an inquiry, a quotation binds the seller to the offered price for the defined validity period.

### Navigation Path
`SAP Easy Access → Logistics → Sales and Distribution → Sales → Quotation → Create`

### T-Code
`VA21`

### Key Inputs

| Field | Value |
|---|---|
| Quotation Type | QT |
| Reference Document | Inquiry No. 10000001 |
| Valid From | (Current Date) |
| Valid To | +15 Days |
| Net Price per Unit | ₹58,000 |
| Total Value | ₹11,60,000 |
| Payment Terms | NT30 (Net 30 Days) |
| Delivery Date | +7 Days |

### Output
- **Quotation Document Number** generated (e.g., 20000001)
- Pricing conditions auto-populated from material and customer master data
- Quotation saved and shared with TechNova Enterprises for acceptance

### Screenshot Reference
`02_VA21_quotation_creation.png`

## Step 3 — Sales Order (VA01)

### What It Is
Once the customer accepts the quotation, a Sales Order (SO) is created. This is the most critical document in the O2C cycle — it is the binding contract between the seller and buyer. All downstream documents (delivery, invoice, payment) are created with reference to this sales order.

### Navigation Path
`SAP Easy Access → Logistics → Sales and Distribution → Sales → Order → Create`

### T-Code
`VA01`

### Key Inputs

| Field | Value |
|---|---|
| Order Type | OR (Standard Order) |
| Reference Document | Quotation No. 20000001 |
| Purchase Order No. | PO-TN-2025-001 (Customer's PO) |
| Requested Delivery Date | (Confirmed delivery date) |
| Quantity | 20 EA |
| Net Value | ₹11,60,000 |

### System Checks Triggered
- **Availability Check (ATP):** System confirms 20 units are available in stock
- **Credit Check:** Customer credit limit verified; order within approved limit
- **Pricing:** Net price, taxes (GST 18%), and total value determined automatically

### Output
- **Sales Order Number** generated (e.g., 30000001)
- Schedule lines confirmed with delivery date
- Order saved and released for delivery processing

### Screenshot Reference
`03_VA01_sales_order.png`

## Step 4 — Delivery Processing (VL01N)

### What It Is
Once the sales order is confirmed, a delivery document is created to initiate the physical fulfillment process. This step involves picking the goods from the warehouse and preparing them for dispatch to the customer.

### Navigation Path
`SAP Easy Access → Logistics → Sales and Distribution → Shipping and Transportation → Outbound Delivery → Create → Single Document → With Reference to Sales Order`

### T-Code
`VL01N`

### Key Inputs

| Field | Value |
|---|---|
| Shipping Point | PE-SHIP-01 |
| Selection Date | Delivery Date from SO |
| Reference Sales Order | 30000001 |
| Picking Quantity | 20 EA |

### Actions Performed
1. Delivery document created referencing Sales Order 30000001
2. Picking completed — 20 units of MAT-PB15 picked from warehouse
3. Delivery note printed for dispatch team

### Output
- **Outbound Delivery Document Number** generated (e.g., 80000001)
- Stock moved to "goods issue" staging area
- Delivery ready for Post Goods Issue in next step

### Screenshot Reference
`04_VL01N_delivery.png`

## Step 5 — Post Goods Issue (VL02N)

### What It Is
Post Goods Issue (PGI) is the point at which legal ownership of the goods transfers from PixelEdge Technologies to TechNova Enterprises. This step reduces inventory in the system, triggers accounting entries in FI (COGS posted), and confirms that the goods have physically left the warehouse.

### Navigation Path
`SAP Easy Access → Logistics → Sales and Distribution → Shipping and Transportation → Outbound Delivery → Change`

### T-Code
`VL02N`

### Key Inputs

| Field | Value |
|---|---|
| Outbound Delivery | 80000001 |
| Actual Goods Issue Date | (Dispatch date) |

### What Happens in the System
- **Inventory reduced** by 20 units of MAT-PB15 in Materials Management (MM)
- **FI accounting entry posted:**
  - Dr. Cost of Goods Sold (COGS)
  - Cr. Finished Goods Inventory
- Delivery status updated to "Goods Issue Posted"

### Output
- Delivery document updated with GI status
- Material document created in MM confirming stock reduction
- Accounting document posted in FI

### Screenshot Reference
`05_VL02N_pgi.png`

## Step 6 — Customer Billing / Invoice (VF01)

### What It Is
After goods have been issued, a billing document (customer invoice) is raised. This is the formal demand for payment sent to TechNova Enterprises. The invoice is generated directly from the delivery document and automatically posts revenue and tax entries in SAP FI.

### Navigation Path
`SAP Easy Access → Logistics → Sales and Distribution → Billing → Billing Document → Create`

### T-Code
`VF01`

### Key Inputs

| Field | Value |
|---|---|
| Reference Delivery | 80000001 |
| Billing Type | F2 (Standard Invoice) |
| Billing Date | (Invoice date) |

### What Happens in the System
- Invoice created for 20 units × ₹58,000 = **₹11,60,000 (base)**
- **GST 18%** applied: ₹2,08,800
- **Total Invoice Value: ₹13,68,800**
- **FI accounting entries posted automatically:**
  - Dr. Accounts Receivable (TechNova Enterprises): ₹13,68,800
  - Cr. Revenue Account: ₹11,60,000
  - Cr. GST Payable (Output Tax): ₹2,08,800

### Output
- **Billing Document Number** generated (e.g., 90000001)
- Invoice PDF generated and dispatched to customer
- Open item created in customer's AR account

### Screenshot Reference
`06_VF01_billing.png`

## Step 7 — Payment Receipt (F-28)

### What It Is
The final step in the O2C cycle. TechNova Enterprises remits payment for Invoice 90000001, and the payment is posted in SAP FI. This clears the open accounts receivable item, completing the revenue cycle for this transaction.

### Navigation Path
`SAP Easy Access → Accounting → Financial Accounting → Accounts Receivable → Document Entry → Incoming Payments`

### T-Code
`F-28`

### Key Inputs

| Field | Value |
|---|---|
| Document Date | Payment receipt date |
| Company Code | PETL |
| Currency | INR |
| Bank Account | PETL-BANK-01 |
| Amount Received | ₹13,68,800 |
| Customer Account | CUST-1001 |
| Reference Invoice | 90000001 |

### What Happens in the System
- **FI accounting entries posted:**
  - Dr. Bank Account (Cash/Bank): ₹13,68,800
  - Cr. Accounts Receivable (TechNova Enterprises): ₹13,68,800
- Open AR item cleared — customer account balance returns to zero
- Payment document created and stored for audit trail

### Output
- **Payment Document Number** generated (e.g., 15000001)
- AR item cleared; customer account settled
- O2C cycle complete for this transaction

`07_F28_payment_receipt.png`

## Document Flow Summary

The complete document chain generated across all 7 steps:

```
Inquiry (VA11) ──► Quotation (VA21) ──► Sales Order (VA01)
                                              │
                                   Delivery Doc (VL01N)
                                              │
                                   Post Goods Issue (VL02N)
                                              │
                                   Billing Doc / Invoice (VF01)
                                              │
                                   Payment Receipt (F-28)
```

You can view this graphically in SAP using:
`VA03 → Environment → Document Flow` (from the Sales Order)
